## 核心定位

**文献库驱动的 AI 写作**（Literature-Grounded AI Writing）—— 它不是"帮你写"，而是确保 AI 生成的每一句话都能追溯到用户上传的真实文献，从根本上解决通用 AI 在学术写作中的引用幻觉问题。

**核心差异化**：基于用户本地文献库的 RAG 生成，引用准确率目标 ≥ 90%（通用 AI 仅 40–60%）。

---

## 功能架构（设计稿）

| 模块 | 核心功能 |
|------|---------|
| 文献管理 | 批量上传 PDF（单次 20 篇），自动解析元数据与全文 |
| 文献管理 | 语义检索、标签分类、跨文献全文搜索 |
| AI 对话 | 基于文献库的 RAG 问答，回答带真实引用标注 |
| AI 对话 | 段落生成、续写、改写（4 种模式）、大纲生成（3 种结构） |
| AI 对话 | 多模型路由（GPT-4o / Claude 3.5 / 本地方案） |
| 编辑器 | TipTap 富文本编辑，支持引用标注节点、AI 提示节点 |
| 编辑器 | 引用格式自动生成（GB/T 7714 / APA / MLA 等 5 种） |
| 编辑器 | 版本历史、导出 Word/LaTeX/PDF（Phase 2 规划） |
| 协作 | 团队项目、文献共享库、评论批注（Phase 3 规划） |

## 技术选型（预研）

| 层级 | 预研选型 | 依据 |
|------|---------|------|
| 前端框架 | React 18 + TypeScript | TipTap 编辑器生态优势 |
| 富文本编辑器 | TipTap (ProseMirror) | 自定义引用节点是硬需求 |
| 状态管理 | Zustand | 中等复杂度，轻量够用 |
| 后端框架 | FastAPI | Python LLM 生态 + 原生异步 |
| 向量数据库 | pgvector | MVP 零运维，预留 Milvus 迁移路径 |
| LLM | GPT-4o / Claude 3.5 / 本地 | 多模型路由，按场景择优 |
| 嵌入模型 | BGE-M3 | 中英双语最优，支持本地部署 |
| 文件存储 | 本地 → OSS | 渐进式架构 |

详见 [技术选型决策记录](技术选型决策记录.md)。
