---

## 一句话定位

面向科研用户的"文献库 + RAG 问答 + 论文编辑"三栏一体化写作平台，以**引用溯源准确率 ≥ 90%**（通用 AI 仅 40–60%）为核心差异化，让 AI 生成的每一句话都有出处。

## 项目速览

| 维度 | 内容 |
|------|------|
| **痛点** | 文献管理（Zotero）、AI 续写（豆包 / Notion AI）、论文编辑三者割裂；通用 AI 生成内容"看似合理却无出处"，引用溯源成功率仅 40–60%，学术场景不敢用。 |
| **方案** | 构建"文献入库 → 智能问答 → 辅助写作"闭环：左栏文献库（多引擎解析 + 向量检索）、中栏 AI 对话（RAG 问答带 `[1][2]` 引用角标）、右栏编辑器（TipTap，选中文本浮动工具栏支持改写 / 扩写 / 精简 / 学术化）。核心机制是**引用验证层 + 结构化 Prompt**。 |
| **效果** | 引用溯源准确率 60% → **≥ 90%**；文献 PDF 解析成功率 85% → **98%**；目标写作时间缩短 30%、工具切换 ≤ 2 次 / 小时。主动砍掉"文献自动推荐"，集中攻克引用溯源这一学术场景真痛点。 |

## 详细项目经历（STAR）

**用户洞察**
- 锁定"引用溯源可信"与"保持个人写作风格"为学术写作两大差异化机会点，而非泛化地做"又一个 AI 写作工具"。
- 竞品拆解：Paperpal 只做编辑不参与生成、Jenni AI 续写强但文献管理弱、沁言学术双栏创新但编辑器弱、Elicit 专注综述不参与写作——市场缺"文献 + AI + 编辑器"三合一。

**产品定位**
- 对比豆包、Notion AI、Zotero，明确"学术上下文 + 引用可信"为核心优势，定位为**文献库驱动的一体化写作工作台**。
- 三维差异化：文献管理（入库即解析）、AI 问答（带出处）、编辑器（原生引用节点）。

**关键决策**
- 独立完成原型与 PRD（V1.1，14 章，含 8 张数据库表、17 个 API、三阶段路线图）。
- **主动砍掉"文献自动推荐"功能**，将研发资源集中到引用溯源准确率这一核心指标。
- 技术选型遵循"MVP 够用不超配 + 预留迁移路径"：pgvector → Milvus、React 18 + TipTap、FastAPI、多模型路由（GPT-4o / Claude 3.5 / 本地）。

**核心指标**
- 引用溯源准确率 ≥ 90%
- 文献解析成功率 ≥ 98%
- 写作时间缩短 30%、工具切换 ≤ 2 次 / 小时
- AARRR 指标体系 + 风险矩阵（老虎 / 纸老虎 / 大象分类）纳入 PRD

**项目成果**

- 构建"文献入库 → 智能问答 → 辅助写作"完整闭环。
- 设计"原文高亮 → 一键定位"溯源功能，AI 回答角标点击可跳转至文献原文。
- 完成从用户洞察、竞品分析、技术决策、工程复盘到高保真可交互原型的全流程交付。

**技术难点攻克**
- **RAG 引用准确率 60% → 90%**：BGE-M3 嵌入模型 + 引用验证层 + 结构化 Prompt，杜绝"幻觉引用"。
- **PDF 解析成功率 85% → 98%**：PyMuPDF → PaddleOCR → Grobid → PdfPlumber 多引擎降级策略，覆盖扫描件 / 双栏 / 公式场景。
- **三栏联动状态一致性**：引用事务模型 + 发布订阅 + 乐观锁，保证文献库、对话、编辑器三端引用状态实时同步。

## 核心差异化高亮

> **引用溯源准确率 ≥ 90%** vs 通用 AI 写作工具 **40–60%**

通用 AI 写作工具在学术场景下"看似合理却无出处"，引用溯源成功率仅 40–60%，导致用户不敢直接采用。本项目通过 **BGE-M3 嵌入 + 引用验证层 + 结构化 Prompt** 三重机制，将引用溯源准确率从 60% 提升至 ≥ 90%，并设计"原文高亮 → 一键定位"溯源闭环，让 AI 生成的每一句话都可回溯到具体文献段落。这是区别于 Paperpal / Jenni AI / 沁言学术 / Elicit 等竞品的核心壁垒。
